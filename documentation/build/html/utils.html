Utils¶

This module provides you with various util functions of BGECore.

Configuration file¶

core.utils.loadGameProperty(name)[source]¶

Loads a property from your config.txt file.

The property is loaded as string, then you use the type name to get the apropiate type. e.j: media.device.volume = float(utils.loadGameProperty("volume"))

Parameters:name (string) – Name of the property to load.
Returns:A string containing the value of the property.

Scene Managment¶

core.utils.getSceneByName(name)[source]¶

Get a scene by its name. Only works with loaded scenes.

Deprecated since version 0.3: Use module.scene_game or module.scene_gui instead.

core.utils.setScene(name)[source]¶

Deletes the game scene (if any) and loads a new one.

Note

The new scene will be initialized on the next logic frame.

Parameters:name (string) – Name of the scene to load. If none no scene will be loaded, but the current scene will be removed.
core.utils.removeScene()[source]¶

Removes the game scene. Same functionality that setScene(None).

Parameters:

  • camera_name (string) – Name of the new active camera.
core.utils.setFilter2D(name, camera, slot)[source]¶

Sets or enables a 2DFilter.

Default aviable GLSL filters: Bloom, Vignetting, ChromaticAberration, FXAA.

Some filters may need special game properties, see: TODO

Parameters:
  • name (string) – Name of the filter.
  • camera (KX_GameObject) – The main camera of the scene (where logic bricks are used).
  • slot (integer) – The slot (Logic Brick Actuator ID) where to apply the filter. Slots are 2DFiletre actuators connected with the Python controller on the main camera. Slots name are prefixed with F, e.j. F0, F1, F2.

core.utils.libLoad(filepath, mode, camera)[source]¶

Loads an blend dinamically.

Blends loaded with this method must have only 1 scene. They also need to have a Python controller conected with an always on module mode with the following string: main.libload. This logic bricks are recomended to be used in an object that clearly is the main object of the scene or a camera if any.

Parameters:
  • filepath (string) – Relative path from the data folder where the blend file is placed.
  • mode – The same mode that will be used on bge.logic.LibLoad(), it’s recomended to use “Scene”.
  • camera (KX_GameObject) – The main camera of the scene (where logic bricks are used).
